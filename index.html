Claude Code

Claude Code Interview Prep — 27 Questions

Master CLI commands, memory system (CLAUDE.md), permissions, MCP integration, CI/CD automation, and team architecture strategies across three difficulty levels.

Project/
  ├── CLAUDE.md     # Team conventions (versioned)
  ├── .claude/
  │   ├── settings.json # Shared permissions
  │   └── CLAUDE.md   # Repo-level instructions
  └── src/
      └── CLAUDE.md   # Folder-specific instructions

Junior Level

9 questions · 20–30 min
  • What is Claude Code vs the web interface?
  • How do you install Claude Code?
  • Three essential slash commands: /help /clear /compact
  • How to write an effective prompt?
  • What happens at the context limit (200k tokens)?
  • How does Claude Code handle project files?
  • Plan mode vs implementation mode?
  • How to stop an operation with Escape / Ctrl+C?
  • How to verify installation with claude --version?

Mid-Level

9 questions · 30–40 min
  • How does the CLAUDE.md memory system work?
  • Configuring team permissions via .claude/settings.json
  • How does Claude Code interact with Git?
  • Difference between /compact and /clear?
  • Using non-interactive (headless) mode with -p
  • How to optimize token consumption?
  • How does Claude Code handle bash errors?
  • Configuring hooks in settings.json
  • How does Claude Code select which model to use?

Senior Level

9 questions · 40–50 min
  • Integrating Claude Code into CI/CD pipelines
  • What is MCP (Model Context Protocol)?
  • Architecting CLAUDE.md memory for 10+ developers
  • Evaluating quality of generated code
  • Using subagents (Task tool) effectively
  • Securing Claude Code in enterprise environments
  • How does Claude Code handle monorepos?
  • Claude Code vs IDE Copilot — when to use which?
  • Team prompt engineering strategy with CLAUDE.md
